#
声明提升

    在js中。如果一个变量先被使用。然后再进行声明和赋值。则不会报错。只是变量的值为undefined。

function myFun(){
	alert(a);
	var a = 1;
}
myFun();

    在上面代码中。a的值为undefined；并不会出现'a is not defined'的报错信息。因为在js中,程序运行时会首先把所有的变量声明操作提升到最开始(这也是推荐将所有用到的变量全部在顶部声明的原因)。赋值操作时间则保持不变。则上面的代码等价于：

function myFun(){
    var a；
	alert(a);
	a = 1;
}
myFun();

先声明了一个变量。但是没有进行赋值操作，然后使用这个变量。之后再进行变量的赋值。

    同时，在函数上，也存在同样的现象。函数定义时常使用声明式与表达式两种。如：


function myFun(){ //声明式
    //code
}

var myFun1 = function(){ //表达式
    //code
}

当函数使用声明式进行定义时。js也会默认将函数定义的时间提前，则在声明式定义函数前进行调用该函数也不会出现问题，但是表达式定义则不会进行声明提升。

myFun();//正常运行

function myFun(){ //声明式
    //code
}

myFun1();//报错

var myFun1 = function(){ //表达式
    //code
}
